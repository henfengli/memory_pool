# 优秀分配器参考优化记录

本文档记录 mempool 参考业界优秀内存分配器进行的每次迭代优化，包含参考来源、设计思想、代码改动和性能数据。

---

## 迭代 1: v1.2.0 — Temporal Cadence（来自 mimalloc）

### 参考来源
- **项目**: [microsoft/mimalloc](https://github.com/microsoft/mimalloc)
- **关键文件**: `src/alloc.c` — `mi_free_block_local()`
- **核心思想**: mimalloc 将 free 路径做到极简（仅 push 到 local free list），合并操作延迟到 alloc 时 free_head 耗尽才触发。这种设计称为 "temporal cadence"——free 和 alloc 在时间上自然交替，合并开销被 alloc 的慢路径自然摊销。

### 问题分析
原来的 `tlc_free` 每次 free 都执行 watermark 检查：
```cpp
if (MP_UNLIKELY(b.local_free_count >= MP_HIGH_WATERMARK)) {
    bucket_collect_local(&b);
}
```
虽然用了 `MP_UNLIKELY` 提示分支预测，但在高频 free 场景下仍有额外的比较+分支指令开销。

### 代码改动
**文件**: `src/tlc.cpp` — `tlc_free()` 函数

**改动**: 删除 watermark 检查，free 路径变为纯粹的链表 push + 原子减 + 统计更新：
```cpp
void tlc_free(TLC* tlc, void* ptr, PageMeta* pm, uint32_t bucket_idx) {
    Bucket& b = tlc->buckets[bucket_idx];
    FreeBlock* blk = static_cast<FreeBlock*>(ptr);
    blk->next = b.local_free_head;
    b.local_free_head = blk;
    b.local_free_count++;
    pm->used_count.fetch_sub(1, std::memory_order_relaxed);
    tlc->stats.free_count++;
    tlc->stats.free_bytes += sc_block_size(bucket_idx);
    // 不再检查 watermark，合并由 alloc 慢路径触发
}
```

合并时机不变——`tlc_alloc` 在 `free_head` 为空时依次尝试 `bucket_collect_local` → `bucket_collect_thread_free` → `bucket_alloc_new_pages`。

### 总结
去掉 watermark 检查后，free 路径的指令数减少，小对象交替 alloc+free 场景提升最为显著（+44%~+46%），free-only 路径也有 7%~16% 提升。Fast path 命中率保持不变（98.46%）。
